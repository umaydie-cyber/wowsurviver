# Azeroth Survivor — 游戏功能与构筑设计文档

> 文档状态：目标设计 v0.2；“当前实现”章节与代码同步，其余章节是后续版本验收规格，不代表已经上线。

> 构建说明：职业 HUD 直接复用职业配置表解析当前职业，避免未使用导入导致 TypeScript `noUnusedLocals` 部署检查失败。

> 交互说明：升级卡片使用位于游戏画布上层的原生 HTML 按钮，不再依赖 Phaser 相机与画布输入坐标换算；悬停或键盘聚焦时卡片高亮，点击时立即应用一次词条、锁定其他选项并恢复战斗。

> 职业选择说明：开局四张职业卡同样使用画布上层的原生 HTML 按钮，鼠标悬停与键盘聚焦均会高亮；点击后锁定所有卡片并进入所选职业的战斗，场景重启时会自动清理界面。

## 1. 产品定位

一句话卖点：**把大型职业 RPG 的职业循环压缩成一局 20 分钟、只操作移动的自动战斗 Build 游戏。**

设计参考来自职业分工、波次构筑与传奇装备改变技能机制三类体验，但职业、名称、美术和数值使用项目自己的原创设定。五层构筑职责必须保持清晰：

| 层级 | 回答的问题 | 设计职责 |
| --- | --- | --- |
| 职业 | 这一局如何战斗？ | 决定初始技能、核心资源、生存方式和站位倾向，不做纯属性皮肤 |
| 技能 | 什么会自动发生？ | 占用自动技能槽，按目标、范围、资源和冷却规则自动触发 |
| 天赋 | 这一局往哪条路线成长？ | 升级随机选择，逐步改变技能形态并锁定专精方向 |
| 装备 | 技能规则如何被改写？ | 提供有取舍的机制变体，而不是只有 `+10 攻击` |
| 套装/神器 | 终局 Build 是什么？ | 10–20 分钟形成的追逐目标，联动多个技能并定义最终画面 |

## 2. 不可违背的设计原则

1. **只操作移动。** 所有主动技能必须自动化；玩家的操作深度来自走位、拾取、升级选择和构筑。
2. **职业先于数值。** 新职业必须拥有独特的“生成资源 → 消耗资源 → 获得攻防收益”循环，不能只修改攻击百分比。
3. **升级必须可感知。** 高阶节点优先改变范围、弹道、触发条件、元素反应或持续方式。
4. **选择存在机会成本。** 装备和天赋可以增强一面、削弱另一面；不让所有选项都成为无脑叠加。
5. **屏幕表现服务 Build。** 玩家应能只看战场就判断这是旋风战、火焰法或宠物流。
6. **先做纵切再扩内容。** 先完成一个职业的完整 20 分钟闭环，再扩到 6 职业和 180 个节点。

### 3.1 基础属性与狂暴战士精通

| 属性 | 当前规则 |
| --- | --- |
| 攻击强度 | 影响攻击技能基础伤害；旋风斩和杀戮命令使用该属性 |
| 法术强度 | 影响法术技能基础伤害；寒冰箭和火球术使用该属性 |
| 速度 | 移动速度属性，基础值 100，按百分比换算为当前物理移速 |
| 护甲 | 通过 `100 / (100 + 护甲)` 降低受到的攻击伤害 |
| 魔抗 | 通过 `100 / (100 + 魔抗)` 降低受到的法术伤害 |
| 全能伤害率 | 按百分比提高所有技能最终伤害 |
| 急速 | 按百分比缩短自动技能释放间隔 |
| 精通 | 影响职业精通技能强度；狂暴战士默认为 25 |
| 经验百分比 | 按百分比提高拾取经验球获得的经验 |

狂暴战士精通为 **精通：战斗专注**：造成伤害后获得战斗专注 Buff，持续造成伤害时每秒获得 `精通 / 5` 数值的额外急速；若 3 秒未造成伤害，Buff 消失并重置累计急速。本 Buff 已在战斗界面显示，文本会展示当前额外急速或未激活状态。

## 4. 目标运行结构

### 4.1 开局

选择职业后获得：1 个基础自动技能、1 个职业资源机制、1 个职业被动。自动技能槽初始 1 个，局内最多 6 个；同名技能升级不占新槽位。

### 4.2 升级四选一

每次升级从可用池生成 4 项，至少覆盖其中 3 类：

- 已有技能升级；
- 当前职业的新技能；
- 已选择专精的天赋节点；
- 通用生存或资源强化。

首次选择某系核心节点即进入该专精。5 分钟前允许出现三个专精的入口；确定专精后，至少 60% 的职业选项来自该系，其他系只出现可兼容的浅层节点。刷新、跳过和封印属于后续局外成长，不进入首个纵切版本。

### 4.3 自动释放判定

每个技能依序检查：`存活 → 冷却完成 → 资源足够 → 存在合法目标/范围 → 自动施放 → 扣除资源 → 进入冷却`。目标策略必须显式配置为最近、最密集、生命最高、随机或玩家周围，不能把目标选择散落在技能代码中。

| 职业 | 初始技能 | 核心资源/循环 | 三系天赋 | 成型画面 |
| --- | --- | --- | --- | --- |
| 狂战士 | 旋风斩：每 3 秒环绕自身造成武器伤害 | 怒气；贴近和受击产怒，高怒延长近战技能 | 狂怒 / 武器 / 守护 | 双斧冲入怪群，最终接近永久旋转 |

每个职业的三个专精各规划 10 个成长节点，即首发内容上限为 `6 × 3 × 10 = 180`。开发顺序不一次性填满：先实现狂战士“狂怒”10 节点的纵切，通过验收后再复制数据结构。

## 6. 专精节点模板

每系 10 节点统一按 `2 个入口 + 3 个塑形 + 3 个联动 + 1 个传奇 + 1 个顶点` 组织，方便内容生产和随机池控制。

### 6.1 狂战士 · 狂怒（首个纵切）

| 层级 | 节点 | 效果 |
| --- | --- | --- |
| 入口 | 扩张利刃 | 旋风斩范围 +20% |
| 入口 | 沸腾怒火 | 每命中 5 个敌人获得 1 怒气 |
| 塑形 | 延长风暴 | 旋风斩持续时间 +1 秒，但移动速度 -8% |
| 塑形 | 双斧 | 每次旋转追加一次 55% 伤害的反向斩击 |
| 塑形 | 血路 | 旋风斩击杀敌人时恢复 1% 最大生命，每秒最多触发 2 次 |
| 联动 | 越战越勇 | 每命中 10 个敌人，当前冷却减少 0.25 秒 |
| 联动 | 狂乱步伐 | 旋转期间可穿过普通敌人，并获得 15% 移速 |
| 联动 | 怒不可遏 | 满怒时自动消耗全部怒气，使下一次旋风斩伤害和范围 +50% |
| 传奇 | 风暴之眼 | 旋风斩中心形成拉拽区域，但外圈伤害降低 20% |
| 顶点 | 永恒刀阵 | 旋风斩永久维持；每秒消耗怒气，怒气耗尽后进入 4 秒疲惫期 |

其他 17 个专精遵循同一规模：火焰强调爆炸连锁，冰霜强调减速—冻结—碎冰，奥术强调充能循环；射击强调穿透暴击，兽王强调宠物协同，生存强调陷阱；痛苦强调 DOT 传染，恶魔强调召唤，毁灭强调慢速高伤；酒仙强调承伤延迟与护盾，踏风强调连击，织雾强调治疗转伤；惩戒强调爆发，防护强调格挡反击，圣光强调光环。具体节点在对应职业进入开发迭代时补齐并单独验收，避免文档先产生 170 个未经玩法验证的占位数值。

## 7. 技能层级

| 层级 | 获取时机 | 示例 | 约束 |
| --- | --- | --- | --- |
| 基础技能 | 开局 | 旋风斩、火球术、自动射击 | 直接表达职业循环，单独就能完成前 2 分钟 |
| 职业技能 | 升级池 | 冰霜新星、醉酿投、召唤猎犬 | 自动触发；需要与职业资源或专精标签联动 |
| 传奇技能 | 10 分钟后事件、神器或套装 | 战吼：持续 30 秒使攻击频率翻倍，120 秒冷却 | 强表现、长冷却或严格触发条件，不做普通升级的数值放大版 |

## 8. 装备与套装

装备词条使用“对象 + 规则变更 + 代价”的结构：

- **灰烬法杖**：火球伤害 +50%，火球尺寸 +80%，但弹速 -35%、基础冷却 +20%。
- **雷霆长刃**：武器命中有 30% 概率产生最多弹射 4 次的闪电链，但物理暴击伤害 -25%。
- **醉月葫芦**：醉酿投留下减伤酒池，但每次额外消耗 1 真气。
- **群狼号角**：宠物击杀可召唤临时幼狼，但玩家本体技能伤害 -20%。

套装只在 15 分钟后进入奖励池；2 件提供流派联动，4 件改变终局循环。例如“风暴行者”2 件使旋风斩命中积累风暴层数，4 件使满层召唤跟随玩家的刀刃龙卷。神器占用独立唯一槽，整局最多获得一件。

## 9. 20 分钟单局节奏

| 时间 | 成长目标 | 内容与验收 |
| --- | --- | --- |
| 0–5 分钟 | 学会职业 | 获得 2–3 个基础技能；5 分钟精英验证核心循环 |
| 5–10 分钟 | 选择专精 | 三系入口出现并逐渐收束；10 分钟 Boss 掉落第一件机制装备 |
| 10–15 分钟 | Build 成型 | 技能产生明确联动，敌人用护甲、速度、远程等特性检验短板 |
| 15–19 分钟 | 追逐终局 | 套装、神器和顶点天赋进入池；强度快速提升但保留生存压力 |
| 19–20 分钟 | 职业试炼 | 最终 Boss 针对职业机制设计，而不是单纯增加血量 |

当前 5 分钟 Boss 只是技术原型；目标版本需将普通局时长改为 20 分钟，并允许开发模式从任意阶段开始测试。

## 10. 系统边界与数据结构

建议按以下边界演进，避免把所有技能继续写入 `GameScene`：

```text
职业系统
├── ClassDefinition：初始属性、资源、初始技能、可用专精
├── SkillDefinition：触发、目标、伤害标签、资源和冷却
├── TalentDefinition：前置、专精、等级、规则修改器
├── EquipmentDefinition：技能机制修改器、代价、稀有度
└── BuildState：本局技能槽、天赋、装备、套装与资源

战斗系统
├── SkillRunner：统一自动施放状态机
├── TargetingSystem：最近、最密集、最高生命等策略
├── EffectSystem：伤害、投射物、范围、召唤和持续效果
├── StatusSystem：Buff、Debuff、DOT、冻结、护盾与承伤延迟
└── EncounterDirector：时间阶段、波次、精英和 Boss
```

内容定义必须是纯数据；运行时效果通过稳定的 effect/modifier ID 组合。UI 只读取定义和 BuildState，不直接修改玩家数值。随机升级生成器必须支持种子，保证 Build 问题可复现。

## 11. 实施顺序与验收

1. **架构拆分**：建立定义、BuildState、自动施放和带种子的升级池；以当前旋风斩纵切为迁移基线。
2. **狂怒纵切**：完成旋风斩、怒气、10 个狂怒节点、2 件机制装备、1 套套装和 20 分钟阶段导演。
3. **战斗能力**：补齐范围伤害、DOT、Buff/Debuff、护盾、召唤和资源消费的自动化测试。
4. **六职业扩展**：按职业逐个增加三系内容；每加入一个职业都要有不同的站位、资源曲线和死亡原因。
5. **内容与表现**：职业选择、Build 面板、技能特效、音效、Boss 试炼和局外解锁。

纵切验收标准：同一狂战士至少能形成“永久旋风”“拉拽爆发”“吸血续航”三种可辨认 Build；20 分钟内至少出现 12 次有意义选择；无输入技能键；同一种子可复现升级选项；最终 Boss 可被三种 Build 击败。

## 12. 当前代码模块

- `Player.ts`：输入、基础属性和经验等级。
- `Enemy.ts`：普通敌人与 Boss 属性、追踪行为。
- `Weapon.ts`：当前旋风斩的自动释放、范围命中、怒气生成、天赋机制与旋转表现；`Projectile.ts` 暂为后续远程职业保留。
- `Spawner.ts`：初始波次、递增波次和 5 分钟原型 Boss。
- `LevelSystem.ts`：狂怒专精的四选一天赋池、前置条件与重复强化；后续补充种子和完整 BuildState。
- `progression.ts`：经验曲线与战斗计时格式化（含单元测试）。
- `UI.ts`：HUD、升级卡片和结算界面。
- `GameScene.ts`：当前系统组装、碰撞、世界绘制和占位纹理。
