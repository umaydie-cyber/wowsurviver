# Azeroth Survivor — 游戏功能文档

> 文档状态：Demo v0.1（与当前实现同步）

## 游戏循环

玩家控制影踪武僧在 3200×3200 的战场移动，角色每 2 秒自动寻找最近敌人并发射火焰之息。野狼持续追踪玩家，死亡后留下经验球；拾取经验升级时游戏暂停并出现强化三选一。存活 5 分钟后生成熔岩巨兽。

## 当前规则

| 系统 | 实现 |
| --- | --- |
| 玩家 | 100 HP、200 移速、10 攻击、等级 1；WASD 八向移动，斜向速度会归一化 |
| 火焰之息 | 2 秒冷却、自动锁定、基础 20 伤害、飞行速度 380 |
| 野狼 | 50 HP、80 移速、5 接触伤害；开局 10 只 |
| 刷新 | 每 30 秒一波，每波数量逐渐增加 |
| 经验 | 野狼死亡掉落经验球，每颗 12 XP；首级需求 40 XP |
| 强化 | +20% 攻击、-15% 攻击间隔、+15% 移速 |
| Boss | 300 秒生成；5000 HP、50 接触伤害、48 移速 |
| UI | HP、等级、经验条、计时、升级弹窗、死亡重开 |

## 模块结构

- `Player.ts`：输入、玩家属性和经验等级。
- `Enemy.ts`：普通敌人与 Boss 属性、追踪行为。
- `Weapon.ts` / `Projectile.ts`：自动选敌、冷却与投射物。
- `Spawner.ts`：初始波次、递增波次和 Boss 计时。
- `LevelSystem.ts`：升级暂停、随机强化及效果。
- `progression.ts`：经验曲线与战斗计时格式化（含单元测试）。
- `UI.ts`：HUD、升级卡片和结算界面。
- `GameScene.ts`：系统组装、碰撞、世界绘制和纹理占位资源。

## 资源替换约定

正式资源按 `assets/player`、`assets/enemy`、`assets/skill`、`assets/ui` 分类。当前纹理键为 `player`、`wolf`、`boss`、`flame` 和 `xp`；加载正式素材时保持纹理键即可不改业务模块。

## 构建与部署

GitHub Actions 使用 Node.js 24 运行测试与生产构建，并通过 Vercel CLI 部署。仓库当前未提交依赖锁文件，因此工作流使用 `npm install` 安装依赖，且不启用 `setup-node` 的 npm 缓存；待提交锁文件后可改回 `npm ci` 与依赖缓存。
